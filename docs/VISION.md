# 層底譚 — 制作方針（ビジョン）

## ゴール

- **継続アップデート**しつつ、**商業プロト**として拡張可能な土台にする。
- 数値・システムより、**読み物としての積み上げ**を優先しつつ、ハクスラの歯応えは保つ。

## 物語・世界の軸

- 主題は **「降り手」という社会**（制度・盟約・記録・燈台・主への関係）。
- 階層帯ごとに **軸語彙を決め、同じ語を再登場させて意味を深める**（`lib/game/lore.ts` の `DESCENDER_SOCIETY_AXIS` と各 `LORE_DESCENDER_*`）。
- 断片的なログで足りる範囲をまず埋め、必要ならイベントや ED で弧を足す。
- **因果イベント**（同じ語の再登場に報酬・分岐を必ず紐づける連鎖）は**採用しない**。運用負荷と説明コストを抑える。

## 推奨プレイ環境（本番想定）

- **PC ブラウザ**を本番相当の体験基準とする。
- **スマホはホーム画面への追加（スタンドアロン表示）**を前提にする（`manifest.webmanifest` / Apple Web App）。
- **LINE 内ブラウザ**は当面の共有用。将来的には最適化の主対象から外してよい（通常ブラウザ・PWA に寄せる）。

## 学習できる難しさ

- 初見で数回は落ちてよいが、**ログの【手がかり】**やボス節目の一文で次の仮説が立つようにする。
- **勝利時**は評価や物語調の締めではなく、**戦闘後の HP/MP を一行で淡々と記録**する。
- 死にすぎるときは数値より、**情報とリズム**を先に整える。

## 更新するときのチェックリスト（短）

1. 新しい探索文は、どの階層帯の **軸語彙**に触れているか。
2. 戦闘変更は **【手がかり】**・**勝利一行**と矛盾しないか。
3. PC と **ホーム画面追加**の両方で一通り触ったか。
