# 配布の「手間」について（作者向け・正直メモ）

## ローカルサーバーは買い手にとって楽か

**「たった 2 ステップ」は、すでに Node.js か Python が入っている人向けの話**に近いです。

| 買い手の状態 | 実際の負担 |
|--------------|------------|
| Node / Python あり・端末に慣れている | コマンド 1 本＋ URL を開く → **本当に 2〜3 ステップ** |
| そういう環境が無い | まず **Node か Python のインストール**から説明を読む必要があり、**ダブルクリック一発より重い** |
| IT に疎い層 | 「サーバー」という語だけで**買いにくく感じる**ことは十分あり得る |

なので、「技術的には簡単」と「心理的に買いやすい」は一致しません。**あなたの直感（買いにくいかも）は妥当**です。

## exe にすると楽になるか / コスパは

- **楽になる可能性**: 「このファイルを実行」に寄せられるので、**説明は短くできる**（ただし SmartScreen やウイルス誤検知で止まる人もいる）。
- **作者のコスト**: Electron / Tauri でラッパーを作る工数、**Windows 用コード署名証明書（年間ドル建てが一般的）**、配布ごとのビルド・テスト。  
  **めんどくさいからやらない**なら、無理に exe に寄せなくてよいです。

## いちばんコスパのよい落とし所（おすすめ）

1. **Web でそのまま遊べる URL（例: Vercel）を、商品説明の目立つ場所に無料で載せる**  
   - 買う前・買った後どちらも **ブラウザを開くだけ**で遊べる層が一気に増える。  
   - **有料 ZIP**は「オフラインで持ちたい人」「更新前の版を残したい人」向けの**特典**として割り切ると、説明のプレッシャーが下がる。

2. **ZIP 同梱は今のまま**＋ **Windows 向け `START.bat`**（Node 入りの PC 向け）で、端末を開かずに済む人を増やす。

3. **exe は需要が見えてから**検討で十分。

## 商品文で書くと誤解が減む例

- 「**ブラウザで URL を開くだけで遊べます**（〇〇に掲載）」
- 「同梱 ZIP は、オフライン用です。**初めての方は URL 版をおすすめ**します」
- 「ZIP はローカルサーバー起動が必要です（`README.txt` または `START.bat`）」

これで「買ったのに何をすればいいか分からない」をかなり減らせます。
