# ダウンロード販売の形（BOOTH 等）— 最短で 100 円から試す

## ブラウザ版 ZIP でよいか / Unity が必要か

- **このリポジトリの形（Next.js を静的書き出しした ZIP）＝ブラウザで遊ぶ版**が、いちばんシンプルです。購入者は **Node か Python でローカルサーバーを 1 回立てる**（`README.txt` に手順あり）。**Unity やインストーラは不要**です。
- **exe を配りたい**場合は後から Electron / Tauri でラッパーを足す選択肢がありますが、工数とコード署名・ウィルス誤検知の話が増えるので、**まずはブラウザ ZIP で十分**というのが同人でも一般的です。

## よくある店の形（BOOTH）

1. **商品ページ**に次を載せる:
   - **キャッチ**（1〜2 行）＋ **スクショ 3〜5 枚**
   - **動作環境**: PC 推奨、Chrome / Edge / Safari 想定、**ローカルサーバー起動が必要**（README 同梱と明記）
   - **同梱物**: ZIP の中身はフォルダ＋`README.txt`＋ゲーム本体（`index.html` と `_next` 等）
2. **価格**: **100 円**は BOOTH で可能。まず「販売の流れ・説明文・更新の仕方」に慣れるのに向いています。
3. **体験版との併用**: 無料 or 投げ銭で **体験版 ZIP**（`npm run pack:demo`）、有料で **フル版 ZIP**（`npm run pack:full`）。同じ作品でも **セーブキーが別**なので混線しにくいです。

## 作者側でやることチェックリスト

- [ ] `npm run pack:full` で ZIP を作り、自分の PC で `README.txt` の手順どおりに起動確認
- [ ] **BGM・フォント・画像**の再配布・商用利用が許されているか確認（自作 or ライセンス確認）→ [ASSETS_AND_RIGHTS.md](./ASSETS_AND_RIGHTS.md)
- [ ] 商品文に **「ローカルサーバー必須」** と **推奨ブラウザ** を書く（誤解で返金トラブルが減る）
- [ ] BOOTH の **特定商取引法に基づく表記**（個人ならテンプレに沿って連絡先・支払い・返品不可など）
- [ ] 価格・ファイル更新したら **版メモ**（商品説明に「v0.1.0」など。`package.json` の version と揃えるとよい）

## ZIP の作り方（開発者向け）

README の「配布用 ZIP（販売）」を参照。要約:

- フル版: `npm run pack:full` → `dist/text-hacks-rpg-full-browser-v0.1.0.zip`（バージョンは例）
- 体験版: `npm run pack:demo`

## 100 円で「売れる」より先に

- まず **1 本でも購入が付くか**・**説明文で誤解が出ないか**を見る段階として有効です。
- 売上より **フィードバック・実績・スクショの改善**を優先してよいです。
